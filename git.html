مقدمة إلى Git

Git هو نظام تحكم بالإصدارات يُستخدم لإدارة التغييرات في الشيفرة المصدرية. يساعدك Git على تتبع الإضافات، التعاون مع فريق، وإدارة إصدارات المشروع.

لماذا Git؟

  • تتبع التاريخ (من يستطيع معرفة ماذا ومتى ولماذا).
  • العمل المتفرع (فروع لميزات جديدة بدون تعطيل الفرع الرئيسي).
  • التعاون (الاندماج، طلبات السحب، ومراجعات الشيفرة).

أوامر أساسية (Quick Start)

افتح الطرفية ثم نفّذ الأوامر التالية داخل مجلد المشروع:

git init
git add .
git commit -m "Initial commit"
# ربط المستودع البعيد (مثال GitHub)
git remote add origin https://github.com/username/repo.git
git branch -M main
git push -u origin main

العمل على فرع جديد

git checkout -b feature/new-ui
# بعد العمل والتأكد
git add .
git commit -m "Add new UI"
git push origin feature/new-ui

السحب والدمج (Pull & Merge)

للحصول على آخر التحديثات من الفرع الرئيسي:

git checkout main
git pull origin main

حل التعارضات

إذا ظهر تعارض عند الدمج، افتح الملفات المتعارضة، حررها، ثم:

git add conflicted-file
git commit -m "Resolve merge conflict"

نصائح سريعة

  • استخدم رسائل التزام واضحة ومختصرة.
  • اعمل على فروع صغيرة ومحددة الغرض.
  • اسحب التغييرات بانتظام لتقليل التعارضات.
